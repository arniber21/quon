## A compiler toolkit for quantum software engineers

Quon brings the language, compiler, backend model, verification seam, and
developer tooling into one Rust workspace. It is built around a typed quantum
frontend, a lowering pipeline through MLIR generic-form IR, and backend
artifacts that can be inspected, simulated, and regression-tested.

## One source, target-specific outputs

```kotlin
fn bell_state(): Circuit<2, 2, 2, Clifford> = circuit {
    H @0 |> CNOT @(0, 1)
}

fn main(): Q<(Bit, Bit)> = run {
    (q0, q1) <- bell_state() @ qreg(2)
    b0       <- measure(q0)
    b1       <- measure(q1)
    return (b0, b1)
}
```

Compile for a fixed gate-model target:

```bash
cargo run -p quonc -- test/verify/bell.qn --emit-qasm
```

Compile for a reconfigurable neutral-atom target:

```bash
cargo run -p quonc -- test/na/bell.qn \
  --target targets/neutral_atom/generic_rna_v0.json \
  --emit-na-schedule \
  --emit-resource-report
```

## Growing toward production-grade workflows

Quon is a living compiler toolkit with a deliberate hardening path: boring
installation, reproducible local CI commands, clearer backend artifact
inspection, stronger benchmark regressions, deeper neutral-atom validation, and
more static invariants connected to backend legality.
